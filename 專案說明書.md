## 1. 專案簡介

### 核心定位

**Document LLM Analysis Agent** 是一個專為**專業文件處理**與**結構化數據分析**設計的 AI 助手。不同於通用型聊天介面（如 Open WebUI），本專案聚焦於「數據驅動」的專業工作流。

### 核心特色

1. **三階段智慧分析工作流**
   - **第一階段**：AI 自動分析資料欄位，提出分析方案建議
   - **第二階段**：使用者採納建議或自訂提問，AI 生成程式碼執行
   - **第三階段**：根據結果，AI 提供「關鍵發現」、「深度洞察」與「行動建議」

2. **專業級文件預處理**
   - 自動處理 Excel 合併儲存格
   - 語義分析分片
   - 多格式支援（PDF/DOCX/TXT/Excel）

3. **混合型 RAG 架構**
   - 本地化 ChromaDB 向量資料庫
   - 支援多語言嵌入模型（BGE-M3）
   - Re-ranking 提升檢索精確度

### 與 Open WebUI 的差異

| 特性 | 本專案 | Open WebUI |
|------|--------|------------|
| **設計核心** | 專業數據工作流（數據驅動） | 通用型聊天介面（對話驅動） |
| **結構化分析** | 提供「建議-執行-判讀」三階段專用工作流 | 以 General Chat 為主 |
| **數據預處理** | 針對 Excel 合併儲存格、正規化資料有專屬邏輯 | 僅基礎文件解析 |
| **UI 風格** | 採用 Architect Intelligence (Material Design 3) 全新設計，優化數據展示與沉浸式閱讀體驗 | 類 ChatGPT 左右側欄配置 |
| **運作模式** | 專注於任務導向的模型調用 | 全面的使用者與模型管理系統 |

## 3. 功能模組

### 3.1 多模態分析（11 種專業模式）

| 模式 | 功能說明 |
|------|----------|
| **對話分析** | 與 AI 進行自然語言對話分析 |
| **摘要生成** | 自動生成文件摘要 |
| **翻譯** | 多語言文件翻譯（中/英/日/韓/法/德/西） |
| **資訊萃取** | 從文件中萃取日期、人名、金額等結構化資訊 |
| **報表生成** | 根據內容生成結構化報表 |
| **風格轉換** | 將內容改寫為指定風格 |
| **文件校對** | 檢查文件中的錯誤 |
| **圖表繪製** | AI 自動繪製圖表 |
| **RAG 文件問答** | 針對長 PDF/Word 提供語義搜索問答 |
| **智慧查詢** | 自然語言提問自動生成 Pandas 程式碼並執行 |
| **統計分析** | 完整的統計分析功能 |

### 3.2 RAG 檢索增強生成

#### 功能特點

- 支援 PDF/DOCX/TXT/Excel 上傳
- 使用 BGE-M3 多語言嵌入模型
- ChromaDB 本地向量資料庫
- 支援 Re-ranking 提升檢索精確度
- Astute RAG：結合內部知識與外部檢索的智慧問答

#### 工作流程

```
文件上傳 → 文件解析 → 分片處理 → 向量化 → 存入 ChromaDB
                                                      ↓
使用者提問 → 向量相似度搜尋 → Re-ranking → 上下文組裝 → LLM 生成回答
```

### 3.3 統計分析功能

#### 描述性統計

- 平均值、中位數、眾數
- 標準差、變異數
- 偏度、峰度
- 百分位數

#### 相關性分析

- Pearson 相關係數
- Spearman 相關係數

#### 推論統計

- **T-檢定**：獨立樣本、成對樣本
- **ANOVA**：單因子變異數分析
- **Shapiro-Wilk**：常態性檢定
- **Mann-Whitney U**：無母數檢定
- **Kruskal-Wallis**：無母數多群組檢定
- **Wilcoxon**：配對樣本無母數檢定
- **Chi-Square**：類別變數獨立性檢定

#### 批次切片過濾

- **多欄位篩選**：支援同時對多個欄位進行條件篩選
- **批次報告產生**：依切片群體自動產生多份分析報告（如：每個部門、每個區域各一份）

#### 機器學習模型

- **線性回歸**：連續數值預測
- **邏輯斯回歸**：二元分類預測
- **Isolation Forest**：ML 異常偵測
- **Prophet 時間序列**：未來趨勢預測（需安裝 `prophet`）

#### 降維與分群

- **PCA**：主成分分析
- **K-Means**：集群分析

#### 資料轉換

- Log 轉換
- 標準化（Z-score）
- One-Hot Encoding
- 正規化
- 缺失值補值
- 標籤編碼

#### SQL 查詢

透過 DuckDB 執行 SQL 查詢，支援複雜資料操作。

### 4.8 LLM 排隊系統

#### 系統概述

針對 GPU RAM 有限的環境，系統提供 LLM 排隊機制，確保資源公平分配：

- **佇列管理**：先進先出（FIFO）排隊機制
- **即時狀態**：顯示目前佇列長度、等待位置
- **自動通知**：輪到時自動開始處理
- **超時保護**：預設 5 分鐘逾時自動釋放

#### 運作流程

```
使用者請求 LLM
    ↓
檢查是否有空閒資源
    ↓
[是] → 直接執行
[否] → 加入佇列等待
    ↓
輪到時自動開始
    ↓
完成後釋放資源
    ↓
通知下一個等待者
```

#### 前端狀態顯示

- **LLM 可使用**：綠色標示，可直接執行
- **排隊中**：琥珀色標示，顯示目前位置
- **LLM 使用中**：藍色標示，顯示等待人數

## 9. 優化建議

### 9.1 架構優化

#### 問題：共用邏輯散落 (✅ 已解決)

原本 `src/services` 與 `backend/app/services` 存在重複邏輯。
**解決方案**：已於重構時將舊版 `src/` 中的核心模組（統計、解釋、模型、報告等）全面遷移至 `backend/app/services`，並統一所有引用路徑，移除 `sys.path.insert`。

#### 問題：前端架構不統一 (✅ 已解決)

原本同時存在 Streamlit（舊版）與 Next.js（新版）兩套前端。
**解決方案**：已於 2026/03/28 完全移除 Streamlit 相關程式碼，確立 Next.js 為唯一前端。

#### 問題：錯誤處理不一致

**建議**：
```python

from fastapi import HTTPException
from typing import Optional

class AppException(Exception):
    def __init__(self, code: str, message: str, detail: Optional[dict] = None):
        self.code = code
        self.message = message
        self.detail = detail or {}

# 全域例外處理器
@app.exception_handler(AppException)
async def app_exception_handler(request, exc):
    return JSONResponse(
        status_code=400,
        content={"code": exc.code, "message": exc.message, "detail": exc.detail}
    )
```

### 9.4 安全性優化

#### 問題：API 無 Rate Limiting (✅ 已解決)

目前 `backend/app/main.py` 已全面導入 `slowapi` 實作 Rate Limiting，預設限制為 `RATE_LIMIT_DEFAULT`（可透過環境變數配置），有效防範 DDoSs 與暴力破解攻擊。

#### 問題：檔案上傳未驗證 (✅ 已解決)

目前 `backend/app/api/upload.py` 已實作嚴格的安全驗證機制，包含：
1. **檔案大小限制**：`MAX_UPLOAD_SIZE = 50MB`
2. **副檔名白名單**：僅允許特定格式 (`.pdf`, `.docx`, `.csv`, `.xlsx` 等)
3. **MIME 型態驗證**：使用 `python-magic` 進行底層 Magic Byte 檢查，防止副檔名偽造攻擊
4. **檔名安全過濾**：使用 `werkzeug.utils.secure_filename` 防止目錄遍歷攻擊

#### 問題：敏感資訊可能寫入 Log

**建議**：
```python
import logging

class SensitiveDataFilter(logging.Filter):
    SENSITIVE_PATTERNS = ['api_key', 'password', 'token', 'secret']
    
    def filter(self, record):
        msg = record.getMessage()
        for pattern in self.SENSITIVE_PATTERNS:
            msg = msg.replace(pattern, '[REDACTED]')
        record.msg = msg
        return True

logger = logging.getLogger()
logger.addFilter(SensitiveDataFilter())
```
